Privacy Policy Effective Date: March 15, 2026 1. Introduction This Privacy Policy describes how MyFitbit (the "App"), a Progressive Web App for my personal use, handles data. The App fetches my own Fitbit data (e.g., activities, heart rate) solely for display in cycling analytics. No third-party user data is collected. 2. Data Collected Fitbit API Data: Only my personal fitness data via OAuth 2.0 (scopes: activity, heartrate). Examples: steps, distance, HR—processed client-side only. Browser Data: Standard web data (IP, cookies, localStorage) for PWA functionality (service worker caching). No analytics trackers. No Other Data: No emails, names, payments, or device IDs stored beyond temporary sessionStorage tokens (8-hour expiry). 3. Data Use All data displays locally in my PWA (e.g., tables synced to Intervals.icu). No server storage, sharing, selling, or external processing beyond Fitbit's API. 4. Data Sharing None with third parties. Fitbit handles all auth/data per their policy. Tokens stored in memory/sessionStorage; cleared on logout/page refresh. 5. Storage and Security Client-side only (localStorage, IndexedDB for offline caching). HTTPS enforced; service worker for offline resilience. No backend servers; Vercel/GitHub Pages hosting. 6. Your Rights (GDPR/CCPA) As sole user (Chris Hancock, France), revoke anytime via Fitbit settings. Data deletion: Clear browser storage or revoke OAuth. 7. Children's Privacy App not directed at children; no COPPA compliance needed. 8. Changes Updates posted here; continued use implies acceptance. 9. Contact Questions? chris@cjhancock.co.uk.
